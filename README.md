# Low-Level Design (LLD)

A comprehensive learning repository covering essential design patterns and LLD interview questions in Java.

## 📚 Design Patterns

*Each pattern folder contains `Good_Code/` (best practices) and `Ugly_Code/` (anti-patterns) implementations*

### Creational Design Patterns
- **Factory Pattern** - Create objects without specifying exact classes
- **Builder Pattern** - Construct complex objects step by step
- **Singleton Pattern** - Ensure only one instance of a class

### Structural Design Patterns
- **Adapter Pattern** - Make incompatible interfaces work together
- **Composite Pattern** - Compose objects into tree structures

### Behavioral Design Patterns
- **Command Pattern** - Encapsulate requests as objects
- **Iterator Pattern** - Access elements sequentially
- **Observer Pattern** - Notify multiple objects of state changes
- **State Pattern** - Alter behavior based on internal state
- **Strategy Pattern** - Define interchangeable algorithms

## 🎯 LLD Interview Questions
- **Tic Tac Toe** - Game logic implementation

## 🏗️ SOLID Design Principles

*Each principle folder contains `Good_Code/` and `Ugly_Code/` implementations*

- **Single Responsibility Principle (SRP)** - A class should have only one reason to change
- **Open/Closed Principle (OCP)** - Open for extension, closed for modification
- **Liskov Substitution Principle (LSP)** - Derived classes should substitute base classes
- **Interface Segregation Principle (ISP)** - Clients shouldn't depend on interfaces they don't use
- **Dependency Inversion Principle (DIP)** - Depend on abstractions, not concrete implementations
